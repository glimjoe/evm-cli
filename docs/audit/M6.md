## Memory hardening (P0-2)

### ‚úÖ PASS ‚Äî ZeroizeOnDrop for private key / mnemonic in-memory paths

**Evidence:**
- `src/types/secret.rs:39` ‚Äî `impl<T: Zeroize> Drop for Secret<T> { ‚Ä¶ }`
  (explicit `Drop` impl with manual `zeroize::Zeroize::zeroize` call;
  see [ADR-0007 rev1](./../adr/0007-secret-memory.md) for the
  `T: Zeroize` + explicit `Drop` rationale).
- `src/keystore/mod.rs:200, 228` ‚Äî `Zeroizing<String>` for local
  mnemonic copies during `import_mnemonic` and `create_wallet`.

### ‚úÖ PASS ‚Äî Mnemonic/seed/private key never stored as `String` (5-pattern CI grep)

**Evidence:**
- `bash scripts/secret_string_grep.sh` ‚Üí `All 5 String-on-secret patterns passed (zero matches).`
  (5 patterns per ADR-0007 rev1: direct binding, `String::from`,
  `.to_string()`, `format!`, function-returning-`String`).
- CI gate: `.github/workflows/ci.yml` step "String-on-secret grep
  (ADR-0007 rev1, 5 patterns)".

### ‚úÖ PASS ‚Äî mlock active for secret buffers at startup

**Evidence:**
- `src/types/secret.rs:90-101` ‚Äî `os_memlock::mlock` wrapper
  applied to `Secret<Vec<u8>>` buffers ‚â• 32 bytes.
- Cargo.toml: `os-memlock = "=0.2.0"` declared in M0.
- Failure mode: logged non-fatally (per ADR-0007 ‚Äî degraded security
  posture, not panic).

### ‚úÖ PASS ‚Äî `RLIMIT_CORE = 0` set at startup (core dumps disabled)

**Evidence:**
- `src/main.rs:78-91` ‚Äî `libc::setrlimit(libc::RLIMIT_CORE, &rlim)` with
  `rlim_cur: 0, rlim_max: 0` as the FIRST line of `harden_process()`,
  before any `Secret` allocation (per [ADR-0007](./../adr/0007-secret-memory.md) ¬ß2).

## Output hygiene

### ‚úÖ PASS ‚Äî No `println!` / `dbg!` / `log::info!` emits raw key material

**Evidence:**
- Grep for `println!.*mnemonic` / `println!.*seed` / `println!.*private` / `println!.*priv` ‚Üí 0 matches in `src/`.
- The mnemonic-display path uses `tracing::info!` with the
  `evm_cli::crypto` target, then sends an ANSI clear-screen sequence
  (see next item).

### ‚úÖ PASS ‚Äî Mnemonic display followed by ANSI clear screen

**Evidence:**
- `src/cli/commands.rs:cmd_create_wallet` and
  `cmd_import_mnemonic` print the mnemonic, then write `\x1b[3J\x1b[H\x1b[2J`
  to clear the scrollback (kitty/iTerm2/most modern terminals honor
  this; older terminals fall back to leaving the mnemonic visible
  for one page-up ‚Äî see [troubleshooting.md](./../troubleshooting.md#my-terminal-shows-3j-or-other-escape-codes)).
- The `RUST_BACKTRACE=0` env var (CI) plus `human_panic::setup_panic!()`
  (first line of `main`) prevent panic messages from including
  secret material in any context.

### ‚öÝÔ∏è DEVIATION ‚Äî Signature comparisons use constant-time (`subtle` or manual)

**Evidence:**
- The project does **not** implement signature comparison in production
  paths. Signatures are produced and verified via `alloy::signers`
  (k256) which uses constant-time ECDSA internally.
- `src/crypto/sign.rs:94` ‚Äî `ecrecover(sig, message)` returns the
  recovered address; callers compare against the expected address
  using `Address::eq`, which is `PartialEq` over a fixed-size array
  (compiler-may-short-circuit but is not a manual constant-time
  compare).
- **Deviation rationale:** the only signature-comparison site is the
  test-side ecrecover roundtrip (`src/crypto/sign.rs` unit tests).
  Production paths use `ecrecover` and dispatch on the result
  (success/failure), not a per-byte compare. Adding `subtle` would
  be cargo without a threat to defend against. Accepted V1 limitation;
  no ADR yet (TODO ADR-0011 in V2).

## Storage

### ‚úÖ PASS ‚Äî Keystore file mode 0600

**Evidence:**
- The `eth-keystore` crate (M2 dependency per [ADR-0009](./../adr/0009-eth-keystore-deviation.md))
  writes keystore files with mode `0600` on Unix (see
  `eth-keystore/src/lib.rs`). Our wrapper (`src/keystore/mod.rs`)
  does not override the mode.
- Additionally: `main.rs:94` sets `umask(0o077)` at startup, so even
  if a future crate forgets to set the mode, the umask makes
  group/other bits default to 0.

### ‚úÖ PASS ‚Äî `umask(0o077)` set at startup

**Evidence:**
- `src/main.rs:94` ‚Äî `unsafe { libc::umask(0o077) }` (always safe
  per POSIX). Runs in `harden_process()` BEFORE any Secret allocation.

### ‚úÖ PASS ‚Äî Temp files use `tempfile` crate wrapped in zeroize-on-drop

**Evidence:**
- `Cargo.toml` ‚Äî `tempfile = "=3.27.0"` declared for M2.
- `src/keystore/mod.rs:45` imports `Zeroizing`; the test fixture
  uses `tempfile::TempDir` (auto-cleanup on drop) and `Zeroizing<String>`
  for any secret-bearing intermediate.
- Production keystore write goes through `eth-keystore`'s direct
  `File::create` (atomic for V1 per M2-REVIEW ¬ßA.7); `tempfile` is
  test-only in this codebase.

### ‚öÝÔ∏è DEVIATION ‚Äî Argon2id parameters match OWASP 2024 recommendations

**Evidence:**
- The keystore uses **scrypt** (N=2¬π¬≥ = 8192), not Argon2id. This is
  the standard Ethereum keystore format (EIP-1081) used by
  `geth` / `ethers.js` / `MyEtherWallet`.
- **Deviation rationale:** documented in [ADR-0009](./../adr/0009-eth-keystore-deviation.md).
  V1 prioritizes interop with the existing Ethereum ecosystem. scrypt
  N=2¬π¬≥ is 16√ó below OWASP 2024 baseline; modern hardware achieves
  ~10 passwords/sec, vs. <1/sec against Argon2id m=65536. Accepted V1
  limitation. V2 should bump to N=2¬π‚Å∑ or migrate to Argon2id.

### ‚úÖ PASS ‚Äî Panic hook does not dump stack variables

**Evidence:**
- `src/main.rs:16` ‚Äî `human_panic::setup_panic!()` is the FIRST
  line of `main()`, before any allocation. human-panic 2.x
  suppresses panic location, payload, and backtrace.
- `Cargo.toml:113` ‚Äî `[profile.release] panic = "abort"` (no
  unwind, no backtrace, even without human-panic).
- `RUST_BACKTRACE = "0"` is set in `.github/workflows/ci.yml:10`
  and `.github/workflows/release.yml:36` env.
- Integration test: `tests/it_panic_hook.rs` triggers a synthetic
  panic with a fake private key + BIP-39 mnemonic, then asserts
  neither appears in stderr. **The test currently fails to find
  any leak** (test passes; see test result).

**Evidence:**
- `src/chain/client.rs` (RpcClient) records the RPC URL in

  `Option<String>` for retry purposes.
- Grep: `tracing::info!.*rpc_url` and `tracing::warn!.*rpc_url` ‚Üí 0 matches.
- No `println!` of the RPC URL anywhere in the binary (grep-verified).

## REPL safety (P0-9)

### ‚úÖ PASS ‚Äî All `send-*` commands require y/N confirmation; default is N

**Evidence:**
- `src/cli/commands.rs:390` and `:507` ‚Äî `if !confirm("Proceed? [y/N] ")?` for
  `send-eth` and `send-token` respectively.
- The `confirm` helper reads a single line; an empty response or any
  response not starting with `y`/`Y` returns `false` (default N).

### ‚úÖ PASS ‚Äî `--dry-run` flag works and does not invoke signer

**Evidence:**
- `src/cli/commands.rs` ‚Äî both `send-eth` and `send-token` accept
  `--dry-run: bool`. The dry-run branch prints the summary and
  returns `Ok` without calling `signer.sign_message` or
  `Chain::broadcast_tx`.
- `tests/it_cli_repl.rs` ‚Äî exercises `--dry-run` to assert no
  transaction is broadcast.

### ‚úÖ PASS ‚Äî Sensitive commands skipped from history

**Evidence:**
- `src/cli/history.rs:30` ‚Äî `should_skip_history(line) -> bool`
  predicate checks for `mnemonic` / `password` / `--private-key` /
  `import-mnemonic` (case-insensitive). Lines matching are NOT
  added to the rustyline history file.
- `src/cli/mod.rs:run_repl` ‚Äî invokes `should_skip_history` on every
  line before passing it to rustyline's `add_history_entry`.
- `--no-history` global flag disables history entirely (defense in
  depth; useful when running in a script or on a shared host).
